# ClickFlow — релизный чеклист

Чеклист для ветки `chore/b1-b2-cleanup` перед мерджем и публикацией.

## 1. Локальная сборка

```bash
./gradlew clean testDebugUnitTest --no-daemon
./gradlew assembleDebug --no-daemon
./gradlew assembleRelease --no-daemon
```

Если `assembleRelease` падает из-за R8/ProGuard — сначала исправить правила, не отключая minify без причины.

## 2. Smoke-test на телефоне

Проверить на реальном устройстве:

- приложение запускается;
- главный экран открывается без crash;
- Accessibility включается в системных настройках;
- Overlay-разрешение включается;
- overlay-метка отображается поверх другого приложения;
- одиночный тап по метке работает;
- остановка overlay-режима работает;
- шаблон изображения сохраняется;
- поиск по картинке находит цель и тапает;
- OCR-поиск по тексту находит цель и тапает;
- сценарий запускается;
- шаги `MARKER`, `PHOTO`, `TEXT`, `WAIT` выполняются;
- панель сценария показывает прогресс;
- кнопка `Стоп` останавливает сценарий;
- после перезапуска приложения настройки и сценарии сохраняются.

## 3. Проверка данных и приватности

- Android Backup выключен в манифесте.
- Privacy Policy актуальна: `docs/PRIVACY_POLICY_DRAFT_RU.md`.
- Data Safety draft актуален: `docs/GOOGLE_PLAY_DATA_SAFETY_RU.md`.
- В приложении нет аналитики/рекламы/сетевого SDK, не отражённого в Data Safety.

## 4. Store assets

Перед публикацией подготовить:

- финальную иконку;
- feature graphic;
- 4–8 скриншотов;
- короткое описание;
- полное описание;
- changelog;
- email поддержки;
- публичную ссылку на Privacy Policy.

## 5. Перед merge в `main`

- Убедиться, что ветка собрана локально.
- Прогнать smoke-test.
- Проверить драфт-PR.
- Не мерджить, если есть compile/runtime ошибки.
