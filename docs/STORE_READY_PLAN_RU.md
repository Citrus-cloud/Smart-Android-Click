## Текущее состояние ветки `chore/b1-b2-cleanup`

Проект очищен от старого simulation/prototype-слоя и приведён ближе к релизной структуре.

### Уже сделано

- Удалены неподключённые пакеты старой симуляции и прототипов: `scenarios/`, `realtap/`, `profiles/`, `audit/`, `backup/`, `diagnostics/`, `safety/`.
- Оставлен рабочий пользовательский поток реальных функций:
  - overlay-метки;
  - реальные тапы через Accessibility;
  - screenshot API Android 11+ через Accessibility для поиска картинки/текста;
  - клик по картинке;
  - клик по тексту через OCR;
  - пользовательские сценарии в `scenario/`.
- Упрощён `ClickFlowViewModel` и главный UI-поток.

  - двухфазный поиск coarse-to-fine;
  - более плотная адаптивная сетка сравнения;
  - диапазон масштаба по умолчанию `0.8–1.2`;
  - шаг масштаба `0.05`.
- Усилены хранилища:
  - нормализация настроек `TextClickStore`;
  - миграция/чистка шаблонов `ImageClickTemplateStore`;
  - нормализация сценариев и активного сценария `ScenarioLibraryStore`.
- Для release-сборки включены:
  - `isMinifyEnabled = true`;
  - `isShrinkResources = true`;
  - базовые ProGuard/R8-правила.
- Android Backup выключен для локальных данных автоматизации.
- Из релизного манифеста убран старый MediaProjection-preview поток.
- Обновлены store-facing тексты Accessibility, Privacy Policy draft и Data Safety draft.

## Что нужно проверить руками перед мерджем

1. Собрать проект локально:

```bash
./gradlew clean assembleDebug --no-daemon
./gradlew testDebugUnitTest --no-daemon
```

2. Если debug проходит — проверить release-сборку:

```bash
./gradlew assembleRelease --no-daemon
```

3. Установить APK на телефон и проверить:

- запуск приложения;
- выдачу Accessibility-разрешения;
- выдачу Overlay-разрешения;
- обычные overlay-метки;
- клик по картинке на Android 11+;
- клик по тексту на Android 11+;
- запуск сценария с шагами `MARKER`, `PHOTO`, `TEXT`, `WAIT`;
- кнопку остановки в overlay/сценарии;
- поведение после перезапуска приложения: настройки и сценарии должны сохраниться.
