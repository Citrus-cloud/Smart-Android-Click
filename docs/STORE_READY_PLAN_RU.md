# ClickFlow — план доведения до магазина

## Что уже готово

- Простая тапалка с метками.
- Плавающие overlay-метки.
- Несколько меток.
- Сохранение настроек.
- Бесконечный режим.
- Задержка старта.
- Аварийная остановка.
- Клик по картинке/иконке.
- Область поиска картинки.
- Multi-scale поиск картинки.
- Постоянный поиск картинки.
- Клик по тексту через OCR.
- Простой сценарий по overlay-меткам.
- GitHub Actions для сборки APK.

## Что улучшить перед публикацией

### 1. Ещё более точный image matching

Текущий matcher уже поддерживает поиск по масштабу. Для магазина позже можно добавить:

- OpenCV;
- подсветку найденной области;
- предпросмотр последнего совпадения;
- более умное сравнение контуров/границ;
- настройку частоты поиска.

### 2. Красивый onboarding

Добавить первый запуск:

1. Что такое ClickFlow.
2. Как включить Accessibility.
3. Как включить overlay.
4. Как поставить первую метку.
5. Как остановить приложение.

### 3. Безопасность

Оставить минимальные, но полезные ограничения:

- аварийная остановка всегда доступна;
- предупреждение для бесконечного режима;
- уведомление, когда идёт поиск картинки/текста;
- простой способ остановить service.

### 4. Release APK

Для магазина понадобится:

- release signing;
- versionCode/versionName;
- changelog;
- privacy policy;
- иконка;
- скриншоты;
- описание.

---

## Моё мнение

ClickFlow уже перешёл из демо в полноценный тестируемый продукт. Главная ценность проекта — не обычный автоклик по координатам, а умные режимы:

- клик по картинке;
- клик по тексту;
- ожидание появления нужного элемента;
- простые сценарии.

Если хорошо отполировать image matching и onboarding, проект можно довести до уровня публичного APK и дальше готовить к магазину.
