# ClickFlow — черновик Google Play Data Safety

Этот документ помогает заполнить раздел **Data Safety** в Google Play Console. Перед публикацией его нужно сверить с фактической финальной сборкой.

## Общий ответ

ClickFlow работает локально на устройстве пользователя и не использует сервер разработчика.

- Данные не продаются.
- Данные не передаются третьим лицам разработчиком приложения.
- Скриншоты, OCR, шаблоны изображений и сценарии обрабатываются локально.
- Android Backup отключён для локальных данных приложения.

---

## Data collection / Сбор данных

### Собирает ли приложение данные пользователя?

Рекомендуемый ответ для текущей версии: **Нет, приложение не собирает данные пользователя на сервер разработчика.**

Пояснение: приложение может локально хранить настройки, сценарии и шаблоны, но они не отправляются разработчику.

---

## Data sharing / Передача данных

### Передаются ли данные третьим лицам?

Рекомендуемый ответ: **Нет.**

ClickFlow не отправляет данные разработчику и не подключает рекламные/аналитические SDK в текущей версии.

---

## Security practices / Практики безопасности

### Шифруются ли данные при передаче?

Если приложение не передаёт пользовательские данные по сети, можно указать, что передача пользовательских данных не выполняется.

### Может ли пользователь запросить удаление данных?

Да, локальные данные можно удалить через настройки Android:

1. Настройки Android → Приложения → ClickFlow.
2. Хранилище → Очистить данные.

Также данные удаляются при удалении приложения.

---

## Разрешения и объяснение для ревью

### Accessibility Service

Используется для пользовательской автоматизации:

- выполнение нажатий, настроенных пользователем;
- запуск пользовательских сценариев;
- локальный снимок экрана на Android 11+ через системный Accessibility screenshot API.

ClickFlow не использует Accessibility для скрытого сбора данных, чтения паролей, платёжных данных или личных сообщений.

### Overlay

Используется для отображения плавающих меток, панели управления и кнопки остановки поверх других приложений.

### Screen capture

Отдельное MediaProjection-разрешение в текущем релизном манифесте не используется. Поиск изображения и текста работает через Accessibility screenshot API на Android 11+ и выполняется локально на устройстве.

## Что нужно добавить перед публикацией

- Финальный email поддержки.
- Ссылка на опубликованную Privacy Policy.
- Проверка, что в финальной сборке нет аналитики, рекламы или сетевых SDK, которые меняют ответы Data Safety.
- Проверка актуальности Accessibility-декларации в Play Console.
