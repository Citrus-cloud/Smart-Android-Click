# ClickFlow — инструкция для тестирования APK

Эта инструкция актуальна для ветки `chore/b1-b2-cleanup` после чистки старого simulation/prototype-слоя и подготовки к релизной сборке.

## 0. Перед установкой

Сначала собрать проект локально:

```bash
./gradlew clean testDebugUnitTest --no-daemon
./gradlew assembleDebug --no-daemon
```

Если debug-сборка и unit-тесты проходят, отдельно проверить release-сборку с R8:

```bash
./gradlew assembleRelease --no-daemon
```

Если `assembleRelease` падает из-за R8/ProGuard — исправить правила в `app/proguard-rules.pro`, не отключая minify без причины.

---

## 1. Первый запуск и разрешения

После установки APK:

1. Открой ClickFlow.
2. Включи **Accessibility** для ClickFlow.
3. Включи разрешение **Поверх других окон / Overlay**.
4. Для режимов фото/текста разреши системный запрос на захват экрана, когда Android его покажет.

Проверить:

- приложение запускается без crash;
- ClickFlow виден в списке Accessibility-служб;
- Accessibility включается и остаётся активным после возврата в приложение;
- Overlay-разрешение включается;
- приложение не просит лишних внутренних consent-экранов старого прототипа.

---

## 2. Overlay-метки и обычные тапы

1. Открой режим меток/overlay.
2. Добавь одну или несколько меток.
3. Перетащи метки на нужные точки.
4. Настрой интервал и количество повторов.
5. Запусти тапы.
6. Останови режим кнопкой Stop.

Проверить:

- метки двигаются;
- можно добавить несколько меток;
- метки отображаются поверх другого приложения;
- тапы проходят в приложение под меткой;
- интервал и количество повторов работают;
- кнопка Stop останавливает работу;
- после перезапуска настройки остаются корректными.

---

## 3. Клик по картинке

Путь:

```text
Расширенные → Настроить картинки
```

Как тестировать:

1. Добавь маленький чёткий шаблон: кнопку, иконку или элемент интерфейса.
2. Настрой порог похожести: обычно 75–85%.
3. При необходимости сузь область поиска.
4. Проверь диапазон масштаба: по умолчанию используется `0.8–1.2`.
5. Выбери разовый или постоянный режим.
6. Запусти поиск и разреши захват экрана.
7. Открой экран, где должен появиться шаблон.

Проверить:

- шаблон сохраняется;
- битые/удалённые шаблоны не ломают список;
- поиск находит шаблон после изменения масштаба/позиции;
- тап идёт в выбранную точку шаблона;
- постоянный режим останавливается;
- нет заметных зависаний на слабом устройстве.

---

## 4. Клик по тексту

Путь:

```text
Расширенные → Настроить текст
```

Как тестировать:

1. Введи текст: `OK`, `Continue`, `Продолжить`, `Забрать` и т.п.
2. Проверь режимы:
   - содержит строку;
   - точное совпадение;
   - игнорировать регистр;
   - постоянный поиск.
3. Запусти режим и разреши захват экрана.
4. Открой приложение с нужным текстом.

Проверить:

- пустой/слишком длинный запрос нормализуется;
- интервал и количество повторов не выходят за безопасные пределы;
- OCR находит русский и английский текст;
- тап выполняется в центр найденной строки/элемента;
- режим корректно останавливается.

---

## 5. Сценарии

Путь:

```text
Расширенные → Открыть сценарий
```

### 5.1. Библиотека сценариев

Проверить:

- создание нового сценария;
- переименование;
- дублирование;
- удаление;
- выбор активного сценария;
- активный сценарий очищается, если его удалить;
- после перезапуска список остаётся валидным.

### 5.2. Редактор сценария

Создать сценарий со всеми типами шагов:

- `MARKER` — тап по координатам;
- `PHOTO` — поиск по шаблону;
- `TEXT` — поиск по OCR;
- `WAIT` — пауза.

Проверить:

- шаги сохраняются;
- порядок шагов не сбивается;
- лимит до 50 шагов соблюдается;
- repeat/interval/wait нормализуются;
- для `PHOTO` и `TEXT` работают политики `SKIP`, `WAIT_RETRY`, `STOP`.

### 5.3. Запуск сценария

1. Убедись, что Accessibility включён.
2. Для фото/текста разреши захват экрана.
3. Запусти сценарий.
4. Проверь плавающую панель прогресса.
5. Останови сценарий кнопкой `■ Стоп`.

Проверить:

- панель показывает название сценария, цикл и текущий шаг;
- во время screenshot-поиска панель не мешает распознаванию;
- бесконечный сценарий останавливается;
- сценарий не продолжает работу после Stop.

---

## 6. Privacy/Store smoke-test

Проверить перед публикацией:

- Android Backup выключен в `AndroidManifest.xml`;
- Privacy Policy draft актуален: `docs/PRIVACY_POLICY_DRAFT_RU.md`;
- Data Safety draft актуален: `docs/GOOGLE_PLAY_DATA_SAFETY_RU.md`;
- приложение не содержит рекламы/аналитики/сетевого SDK, не описанного в Data Safety;
- Accessibility-описание в системных настройках соответствует реальной функции приложения.

---

## 7. Что особенно проверить на разных устройствах

- Android 10/11/12/13/14;
- Xiaomi/MIUI или HyperOS;
- Samsung One UI;
- Pixel/чистый Android;
- поведение Accessibility после перезагрузки;
- работа Overlay поверх сторонних приложений;
- стабильность после 10–20 минут непрерывной работы;
- потребление батареи в постоянном поиске картинки/текста.
