# ClickFlow — инструкция для тестирования APK

## 1. Первый запуск

После установки APK появится короткий onboarding:

1. Что такое ClickFlow.
2. Зачем нужен Accessibility.
3. Зачем нужен Overlay.
4. Где обычные метки.
5. Где расширенные функции.
6. Как остановить приложение.

После onboarding:

1. Включи **Accessibility** для ClickFlow.
2. Включи разрешение **Поверх других окон / Overlay**.
3. Вернись в приложение.

Эти разрешения Android нужны для реальных тапов и плавающих меток. Внутренние лишние согласия убраны.

---

## 2. Обычная тапалка

1. На главном экране поставь одну или несколько меток.
2. Настрой таймер.
3. Настрой количество повторов или включи бесконечный режим.
4. Нажми **Запустить**.

Проверить:
- метки двигаются;
- метки сохраняются после перезапуска;
- таймер сохраняется;
- повторы сохраняются;
- кнопка **Стоп** останавливает работу;
- **Аварийная остановка** сразу прерывает цикл.

---

## 3. Метки поверх экрана

1. Нажми **Метки поверх**.
2. Появится панель ClickFlow и плавающие метки.
3. Добавь метки через **+**.
4. Перетащи метки на нужные точки.
5. Нажми **Старт** на overlay-панели.

Проверить:
- каждая метка двигается отдельно;
- можно добавить до 5 меток;
- можно удалить метку;
- overlay-метки сохраняются;
- тапы проходят в приложение под меткой.

---

## 4. Клик по картинке

Путь:

```text
Расширенные → Настроить картинки
```

Как тестировать:

1. Добавь маленькую картинку кнопки или иконки.
2. Настрой порог похожести: обычно 75–85%.
3. Если знаешь, где картинка появится, сузь область поиска.
4. Если картинка на экране может быть больше/меньше, настрой масштаб поиска.
5. Выбери режим:
   - один тап и стоп;
   - искать постоянно.
6. Нажми **Запустить**.
7. Разреши захват экрана.
8. Открой приложение, где должна появиться эта картинка.

Если ClickFlow найдёт похожий шаблон, он тапнет в выбранную точку.

Совет: лучше использовать маленькие чёткие шаблоны — например только саму кнопку или иконку, а не весь экран.

---

## 5. Клик по тексту

Путь:

```text
Расширенные → Настроить текст
```

Как тестировать:

1. Введи текст кнопки, например:
   - `OK`
   - `Continue`
   - `Продолжить`
   - `Забрать`
2. Выбери режим:
   - содержит строку;
   - точное совпадение;
   - игнорировать регистр;
   - искать постоянно.
3. Нажми **Запустить клик по тексту**.
4. Разреши захват экрана.
5. Открой приложение, где есть нужный текст.

Когда OCR найдёт строку, ClickFlow тапнет в центр найденного текста.

## 6. ✦ Премиум-режим «Сценарии»

Путь:

```text
Расширенные → Открыть сценарий
```

### 6.1. Экран-замок

При первом входе открывается премиум-замок с описанием возможностей.

Проверить:
- показывается градиентный заголовок и список преимуществ;
- кнопка **✦ Разблокировать Премиум** открывает билдер;
- после перезапуска доступ остаётся разблокированным;
- кнопка **Назад** возвращает в приложение.

### 6.2. Библиотека сценариев

1. Нажми **+ Новый сценарий**.
2. Проверь дублирование, удаление и переключение активного сценария.
3. Активный сценарий помечен бейджем.

### 6.3. Редактор сценария

1. Задай название сценария.
2. Настрой повтор всего сценария: число циклов или ∞.
3. Добавь шаги разных типов:
   - **◎ Метка** — тап по координатам X/Y;
   - **🖼 Фото** — поиск по сохранённому шаблону или загрузка нового;
   - **🔤 Текст** — поиск по тексту (OCR);
   - **⏱ Пауза** — задержка.
4. Для каждого шага настрой повтор и интервал.
5. Для фото/текста выбери политику **«если не найдено»**: пропустить / подождать и повторить / стоп.
6. Перемещай шаги стрелками ↑/↓, дублируй и удаляй.

Проверить:
- шаги сохраняются и порядок не сбивается после перезапуска;
- координаты метки можно подобрать в режиме метки;
- фото-шаблоны из раздела «Настроить картинки» доступны в шаге Фото.

### 6.4. Запуск и панель прогресса

1. Убедись, что Accessibility включён (для фото/текста — и захват экрана).
2. Нажми **▶ Старт** на сценарии.
3. Появится плавающая панель прогресса.

Проверить:
- панель показывает название сценария, текущий цикл и шаг;
- кнопка **■ Стоп** на панели останавливает сценарий;
- во время скриншота панель не попадает в поиск (нет ложных срабатываний по самой панели);
- бесконечный цикл корректно останавливается.

---

## 7. Что особенно проверить перед релизом

- Android 10+
- Accessibility включается и остаётся активным.
- Overlay появляется поверх других приложений.
- MediaProjection не падает после разрешения.
- Image click не даёт слишком много ложных совпадений.
- OCR работает на русском и английском тексте.
- Сценарии корректно выполняют цепочку и останавливаются кнопкой Стоп.
- Бесконечные режимы останавливаются кнопками Stop.
- Приложение не зависает после 10–20 минут работы.
