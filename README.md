# ClickFlow Android — умный автокликер

> Полный концепт-документ проекта. Здесь собрано всё: что это за продукт, какой путь он прошёл, что уже готово, что он умеет и как им пользоваться от установки до сложных сценариев. Этого документа достаточно, чтобы любой человек, агент или нейросеть с нуля поняли проект целиком.

---

## 1. Что это за проект

**ClickFlow** — это минималистичный умный автокликер для Android. Приложение само нажимает (тапает) по экрану за пользователя: по фиксированным точкам, по найденной на экране картинке или по найденному тексту. Из этих действий можно собирать целые **сценарии** — цепочки шагов, которые выполняются автоматически и могут повторяться по кругу.

Главная идея — дать пользователю мощную автоматизацию тапов **без root**, используя только штатные системные механизмы Android (Accessibility, Overlay, MediaProjection).

Типичные задачи, которые решает ClickFlow:

- автоматически нажимать одну или несколько точек по таймеру;
- кликать по кнопке/иконке, как только она появляется на экране (поиск по картинке);
- кликать по кнопке с определённым текстом («OK», «Продолжить», «Забрать» и т.д.) через распознавание текста;
- собирать сложные цепочки действий в сценарий и запускать их в один тап.

**Платформа:** Android 8.0+ (minSdk 26), целевой Android 14 (targetSdk 34). Интерфейс на Jetpack Compose, Material 3.

## 2. Статус проекта

**Основная разработка завершена.** Все ключевые режимы реализованы, протестированы на уровне модели юнит-тестами и задокументированы. Текущая версия — `0.2.0-beta` (`versionCode = 2`).

## 3. История проекта: путь от простой тапалки до премиум-сценариев

Ниже — хронология развития, от самого начала до текущего состояния.

### Этап 0. Базовая тапалка
- Приложение начиналось как простой автокликер по одной или нескольким **меткам** (фиксированным точкам экрана).
- Добавлены таймер, количество повторов, бесконечный режим и аварийная остановка.
- Реализовано сохранение настроек и позиций меток между запусками.

### Этап 1. Плавающие overlay-метки
- Метки вынесены **поверх других приложений** (overlay), чтобы расставлять точки прямо на нужном экране.
- Каждая метка перетаскивается отдельно; до 5 меток одновременно.
- Overlay-метка стала главной «фишкой» продукта — это её ключевое преимущество.

### Этап 2. Клик по фото
- Добавлен поиск картинки/иконки на экране через захват экрана (MediaProjection) и сопоставление шаблона.
- Настройки: порог похожести, область поиска, диапазон масштаба, разовый или постоянный режим, интервал и число повторов.

### Этап 3. Клик по тексту
- Добавлено распознавание текста на экране (OCR через ML Kit) и тап по найденной строке.
- Режимы: «содержит строку» / «точное совпадение», игнорирование регистра, постоянный поиск.

### Этап 4. Чистка и стабилизация
- Из основного пользовательского потока убраны лишние внутренние согласия и демо-чеклисты.
- Исправлены ключевые баги: метка не реагировала на тап, мерцание, ложные тосты о невключённом Accessibility.
- Маркер сделан крупнее, полупрозрачным, с удобным перетаскиванием касанием.
- Панель параметров переработана: компактная, без лишних рамок, с удобной ручкой для перетаскивания.

### Этап 5. Премиум-режим «Сценарии» (6 фаз)
Самая крупная веха. Реализован полноценный конструктор сценариев:

- **Фаза 1 — Модель данных.** Чистая Kotlin-модель сценария: `Scenario`, `ScenarioStep`, типы шагов (`MARKER`, `PHOTO`, `TEXT`, `WAIT`), политика «если не найдено» (`SKIP`, `WAIT_RETRY`, `STOP`). Хранилище библиотеки сценариев в SharedPreferences (JSON).
- **Фаза 2 — Движок.** Сервис исполнения сценария (`ScenarioEngineService`), который последовательно выполняет шаги, использует поиск по фото, OCR и реальные тапы через Accessibility.
- **Фаза 3 — Конструктор (UI).** Премиальный экран сборки сценария на Compose: библиотека сценариев, редактор шагов, настройки повторов и циклов.
- **Фаза 4 — Панель прогресса.** Плавающая панель поверх экрана во время работы: текущий сценарий, цикл, шаг и кнопка «■ Стоп».
- **Фаза 5 — Премиум-гейт.** Экран-замок перед сценариями с описанием возможностей и кнопкой разблокировки; флаг разблокировки сохраняется, заложен хук под будущую подписку.
- **Фаза 6 — Тесты и документация.** Юнит-тесты модели сценария и полное обновление документации (этот README и инструкция по тестированию).

Параллельно велась подготовка вспомогательных материалов: инструкция по тестированию, план вывода в магазин, черновик политики конфиденциальности.

## 4. Возможности (что продукт умеет)

- **Обычная тапалка** по одной или нескольким меткам с таймером, повторами и бесконечным режимом.
- **Плавающие overlay-метки** — расстановка точек прямо поверх любого приложения.
- **Клик по фото** — тап по картинке/иконке, найденной на экране.
- **Клик по тексту** — тап по строке, распознанной через OCR.
- **Премиум-сценарии** — цепочки из шагов разных типов с индивидуальными настройками и зацикливанием.
- **Плавающая панель прогресса** во время выполнения сценария.
- **Аварийная остановка** в любой момент.
- Сохранение всех настроек, меток, шаблонов и сценариев между запусками.

## 5. Установка

### Сборка из исходников

```bash
./gradlew testDebugUnitTest --no-daemon   # прогнать юнит-тесты
./gradlew assembleDebug --no-daemon        # собрать debug APK
```

Готовый APK появится здесь:

```text
app/build/outputs/apk/debug/app-debug.apk
```

### Установка APK на телефон

1. Перенеси `app-debug.apk` на телефон (или скачай из артефактов сборки).
2. Открой файл и разреши установку из этого источника, если Android попросит («Установка неизвестных приложений»).
3. Установи и открой ClickFlow.

## 6. Разрешения — как всё включить (обязательный шаг)

ClickFlow работает только через штатные разрешения Android. Без них тапы и метки поверх экрана не будут работать. Включить нужно следующее.

### 6.1. Accessibility (Специальные возможности) — для реальных тапов
Это главное разрешение: именно через службу специальных возможностей приложение выполняет настоящие нажатия.

1. Открой **Настройки телефона → Специальные возможности (Accessibility)**.
2. Найди в списке **ClickFlow** (обычно в разделе «Скачанные приложения» / «Установленные службы»).
3. Включи службу и подтверди системное предупреждение.
4. Вернись в ClickFlow.

Если тапы не проходят — проверь, что служба осталась включённой (некоторые прошивки отключают её после перезагрузки).

### 6.2. Overlay (Поверх других приложений) — для меток поверх экрана
Нужно, чтобы метки и панели отображались поверх других приложений.

1. Открой **Настройки → Приложения → ClickFlow → Поверх других приложений** (или «Отображение поверх других окон»).
2. Включи разрешение для ClickFlow.
3. Вернись в приложение.

На некоторых телефонах путь: **Настройки → Специальные возможности / Конфиденциальность → Поверх других приложений**.

### 6.3. MediaProjection (Захват экрана) — для фото и текста
Нужно для поиска картинки и распознавания текста на экране. Запрашивается автоматически в момент запуска соответствующего режима.

1. Запусти режим «Клик по фото», «Клик по тексту» или сценарий с такими шагами.
2. Android покажет системный запрос **«Разрешить захват экрана»** — нажми «Начать сейчас» / «Разрешить».
3. Захват используется только для анализа экрана внутри приложения.

### 6.4. Уведомления (Android 13+)
При первом запуске разреши уведомления, чтобы видеть статус работы и иметь стабильный фоновый режим.

## 7. Полное руководство по использованию

### 7.1. Обычная тапалка (метки по точкам)

1. На главном экране добавь одну или несколько меток.
2. Настрой **таймер** (интервал между тапами).
3. Укажи **количество повторов** или включи **бесконечный режим**.
4. Нажми **Запустить**.
5. Останови кнопкой **Стоп** или **Аварийной остановкой**.

Метки, таймер и повторы сохраняются между запусками.

### 7.2. Метки поверх экрана (overlay)

Главная фишка продукта — расставлять точки прямо на нужном экране.

1. Нажми **Метки поверх**.
2. Появятся плавающая панель ClickFlow и сами метки.
3. Добавляй метки кнопкой **+** (до 5 штук), перетаскивай их касанием на нужные точки.
4. Лишнюю метку можно удалить.
5. Нажми **Старт** на overlay-панели — приложение начнёт тапать по точкам под метками.
6. Панель параметров можно перетаскивать за верхнюю ручку.

Overlay-метки и их позиции сохраняются.

### 7.3. Клик по фото

Путь: **Расширенные → Настроить картинки**.

1. Добавь маленькую чёткую картинку — например только саму кнопку или иконку (не весь экран).
2. Настрой **порог похожести** (обычно 75–85%).
3. При необходимости **сузь область поиска**, если знаешь, где появится картинка.
4. Если элемент может менять размер — настрой **диапазон масштаба**.
5. Выбери режим: **один тап и стоп** или **искать постоянно** (с интервалом и числом повторов).
6. Нажми **Запустить** и разреши **захват экрана**.
7. Открой приложение, где должна появиться картинка.

Как только шаблон найден на экране, ClickFlow тапает в выбранную точку картинки.

**Совет:** чем меньше и контрастнее шаблон, тем меньше ложных срабатываний.

### 7.4. Клик по тексту (OCR)

Путь: **Расширенные → Настроить текст**.

1. Введи искомый текст кнопки: `OK`, `Continue`, `Продолжить`, `Забрать` и т.п.
2. Выбери режим сопоставления:
   - **содержит строку** или **точное совпадение**;
   - **игнорировать регистр**;
   - **искать постоянно**.
3. Нажми **Запустить клик по тексту** и разреши **захват экрана**.
4. Открой приложение, где есть нужный текст.

Когда OCR находит строку, ClickFlow тапает в центр найденного текста. Работает с русским и английским текстом.

### 7.5. Премиум-сценарии (главный продвинутый режим)

Сценарий — это собранная пользователем цепочка шагов, которая выполняется по порядку и может повторяться по кругу. Это самый мощный режим: он объединяет метки, фото, текст и паузы в единую автоматизацию.

Путь: **Расширенные → Открыть сценарий**.

#### Шаг 1. Экран-замок (Премиум)
При входе открывается премиум-экран с описанием возможностей. Нажми **«✦ Разблокировать Премиум»**.
- Сейчас, на этапе тестирования, разблокировка **бесплатна** и открывает все функции.
- Подписка/оплата появится позже; флаг разблокировки сохраняется на устройстве.

#### Шаг 2. Библиотека сценариев
После разблокировки открывается список твоих сценариев.
- **+ Новый сценарий** — создать новый.
- Для каждого сценария: **Старт**, **Изменить**, сделать **Активным**, **Дублировать**, **Удалить**.
- Активный сценарий помечается бейджем. Встроенного каталога нет — все сценарии создаёшь сам.

#### Шаг 3. Редактор сценария
1. Задай **название** сценария.
2. Настрой **повтор всего сценария**: фиксированное число циклов или **∞ (бесконечно)**.
3. Добавляй шаги кнопками внизу:
   - **◎ Метка** — тап по координатам X/Y (координаты можно подобрать в режиме метки).
   - **🖼 Фото** — поиск по сохранённому шаблону или загрузка нового фото прямо из шага.
   - **🔤 Текст** — поиск по тексту (OCR) с настройками «часть строки» и «игнорировать регистр».
   - **⏱ Пауза** — задержка на заданное число миллисекунд.
4. Для каждого шага настраивается **повтор** и **интервал** между повторами.
5. Для шагов **Фото** и **Текст** выбирается политика **«если не найдено»**:
   - **Пропустить** — идти дальше;
   - **Подождать и повторить** — с настройкой времени ожидания и числа попыток;
   - **Стоп** — остановить сценарий.
6. Шаги можно **перемещать** стрелками ↑/↓, **дублировать** и **удалять**. До 50 шагов в сценарии.

#### Шаг 4. Запуск и панель прогресса
1. Убедись, что включён Accessibility (а для фото/текста — ещё и разрешён захват экрана).
2. Нажми **▶ Старт** на сценарии.
3. Поверх экрана появится **плавающая панель прогресса**: название сценария, текущий цикл и шаг, кнопка **■ Стоп**.
4. Останови сценарий кнопкой на панели в любой момент.

Во время скриншота панель прогресса автоматически скрывается, чтобы не мешать поиску по фото/тексту.

## 8. Архитектура и структура кода

Основные пакеты (`app/src/main/java/com/clickflow/android/`):

- `overlay/` — плавающие overlay-метки и их сервис (`FloatingTapperOverlayService`).
- `imageclick/` — клик по фото: хранилище шаблонов, сервис, экран настройки, сопоставление шаблонов.
- `textclick/` — клик по тексту: хранилище, сервис, экран настройки (OCR через ML Kit).
- `permissions/` — служба специальных возможностей (`ClickFlowAccessibilityService`): реальные тапы и захват экрана.
- `scenario/` — премиум-сценарии:
  - `ScenarioModel.kt` — чистая Kotlin-модель (`Scenario`, `ScenarioStep`, `StepType`, `NotFoundPolicy`), покрыта юнит-тестами.
  - `ScenarioLibraryStore.kt` — хранение библиотеки сценариев (JSON в SharedPreferences).
  - `ScenarioActivity.kt` — премиум-конструктор и экран-замок (Compose).
  - `ScenarioEngineService.kt` — исполнение сценария + плавающая панель прогресса.
  - `ScenarioPremium.kt` — флаг разблокировки и хук под будущую подписку.
- `core/`, `ui/` — общая навигация, тема и экраны.

Тесты: `app/src/test/java/com/clickflow/android/scenario/ScenarioModelTest.kt` — JVM-юнит-тесты модели сценария.
